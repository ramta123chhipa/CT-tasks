# Superstore Data Analysis

---

##  Project Description
This project involves analyzing a retail Superstore dataset using SQL. The goal is to perform data cleaning, normalization, and advanced analytics to extract meaningful business insights such as top customers, sales trends, and profitability analysis.

---

##  Dataset
- Source: Superstore dataset (CSV format)
- Imported using: MySQL Workbench Table Data Import Wizard
- Raw table: `sample_superstore`

---

##  Tools & Technologies Used
- MySQL Workbench
- SQL (Structured Query Language)
- CSV Dataset
- Git & GitHub

---

##  Database Structure
- `sample_superstore` → Raw staging table
- `orders` → Cleaned transactional table
- `customers` → Customer data ( normalized)
- `products` → Product data ( normalized)

---

## SQL Concepts Used

###  Basic SQL
- SELECT, WHERE, ORDER BY

###  Aggregations
- SUM(), AVG(), COUNT(), GROUP BY

###  Subqueries
- Above average sales
- Highest order per customer

###  CTE (Common Table Expressions)
- Total sales per customer
- Aggregated analysis queries

###  Window Functions
- ROW_NUMBER()
- RANK()
- DENSE_RANK()

---

##  Business Insights Generated
- Top customers based on total sales
- Low-performing customers
- Single-order customers
- Above-average sales transactions
- Category-wise and region-wise performance analysis
- Impact of discounts on profit

---

##  Key Learnings
- Real-world data requires cleaning before analysis
- SQL is powerful for business decision-making
- Window functions help in advanced analytics
- CTE improves query readability and structure
- Data normalization improves database efficiency
